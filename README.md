# 密聊 · 端對端加密聊天室

一個**端對端加密（E2EE）**、**零知識中繼**、**可安裝到手機**的私密聊天室。
單一 Node 程式即可執行，**零外部相依套件**（連 WebSocket 都是用 Node 內建自幹的），程式碼可完整審閱。

> 設計目標：伺服器營運者與網路中間人都**只看得到密文**，藉此防止聊天內容外洩。

---

## ✨ 功能

| 類別 | 功能 |
| --- | --- |
| 加密 | AES‑256‑GCM 逐則訊息加密、HKDF 金鑰衍生、訊息完整性驗證（竄改會被拒收） |
| 分享 | 一條連結或 **QR 碼**即分享整個房間；**金鑰只放在網址 `#` 之後，永不送到伺服器** |
| 頻道 | 同一條連結 = 同一個頻道，多人即時聊天、線上人數顯示 |
| 手機 | 響應式 RWD、PWA 可「加入主畫面」離線啟動、iOS 安全區域適配 |
| 隱私 | 無需帳號/手機/Email、伺服器不保存訊息、嚴格 CSP、無第三方請求與追蹤 |
| 安全增能 | **前向保密**（群組寄件金鑰 + 一次性 ECDH，過往訊息不可回溯解密）、訊息自動消失、一鍵清除本機資料（panic wipe）、**安全驗證碼**（防連結被竄改）、**本機密碼鎖**、**訊息大小填充**（降低長度 metadata 外洩） |
| 檔案 | 端對端加密的圖片/檔案傳送（含縮圖、上限 2MB） |

## 🚀 快速開始

需求：**Node.js 18+**（本專案在 Node 24 測試通過）。

### 最簡單：一鍵啟動（Windows）

直接點兩下專案資料夾裡的 **`一鍵啟動聊天室.bat`**：

1. 自動啟動伺服器 + Cloudflare 分享隧道
2. 等隧道生效後**自動打開瀏覽器**（手機/他人可用的 HTTPS 網址）
3. 按「建立加密聊天室」→ 按 🔗 複製邀請連結傳給對方，完成

> 使用期間保持該視窗開啟；關閉視窗＝關閉聊天室。
> 若未安裝 `cloudflared`，會顯示安裝指令（`winget install --id Cloudflare.cloudflared --source winget`），並先開啟僅限本機的 localhost 版。
> 想放桌面：對 `.bat` 按右鍵 →「傳送到」→「桌面（建立捷徑）」。

### 進階：固定網址（推薦，一次設定終身受用）

預設的 Cloudflare 快速隧道**每次啟動網址都不同**，且部分家用路由器 DNS 對新網域反應慢。
點兩下 **`設定固定網址.bat`** 走一次引導（註冊免費 ngrok 帳號 → 貼 authtoken → 貼你的專屬網域），
之後「一鍵啟動聊天室.bat」的網址就**固定不變**（例：`https://happy-cat-1234.ngrok-free.dev`），朋友可直接存書籤。

- 設定會寫入 `share.config.json`（只存網域名稱，不含任何祕密；authtoken 由 ngrok 存在你的使用者設定中）。
- 想改回 Cloudflare 隨機網址：把 `share.config.json` 刪掉即可。
- 朋友第一次開 ngrok 免費網址會看到一頁 ngrok 提示，按「Visit Site」即可進入（每個瀏覽器只出現一次）。

### 手動：僅本機執行

```bash
# 於專案根目錄
node server/server.js
```

開啟 <http://localhost:8787> → 按「建立加密聊天室」。（localhost 只有這台電腦能用；要分享給別人請用上面的一鍵啟動或見下節。）

> ⚠️ **Web Crypto 只在安全環境可用**：`http://localhost` 或 `https://` 才行。
> 用純 `http://` 的 IP（例如 `http://192.168.x.x`）開啟會無法加密——請見下方「分享給手機/他人」。

重新產生 PWA 圖示（可選）：`node tools/gen-icons.js`

## 📱 分享給手機或其他人（需要 HTTPS）

因為瀏覽器規定，加密功能在遠端裝置上**必須走 HTTPS**。最省事的三種方式：

### 1) Cloudflare Tunnel（最快，免網域、免開防火牆）

**一鍵版（推薦）**——會自動同時啟動伺服器與隧道，並印出可用網址：
```bash
npm run share
```
> 若尚未安裝 `cloudflared`，腳本會提示各平台安裝方式（Windows：`winget install --id Cloudflare.cloudflared`）。

**手動版**：
```bash
# 另開一個終端機，先啟動 node server/server.js，再執行：
cloudflared tunnel --url http://localhost:8787
```
兩種方式都會得到一個 `https://xxxx.trycloudflare.com` 網址，手機直接開就能用。
（隧道商也只看得到**密文**，E2EE 依然成立。）

### 2) ngrok
```bash
ngrok http 8787
```

### 3) 自有網域 + Caddy（正式部署，自動 HTTPS）
見 [DEPLOY.md](DEPLOY.md)。

在手機瀏覽器開啟後，選單選「加入主畫面」即可像 App 一樣使用。

## 🔐 安全模型（請務必理解）

### 受保護的（威脅模型內）
- **伺服器營運者**：只看到路由用的 `channelId`、（已填充成固定級距的）訊息大小與時間，**看不到金鑰或內容**。訊息在加密前會先於密文內填充到大小級距，短訊息彼此的長度無法被區分。
- **網路監聽者 / 隧道商 / 反向代理**：只看到 TLS 之下的密文。
- **訊息竄改**：AES‑GCM 驗證標籤會讓竄改或錯誤金鑰的訊息**直接被拒收**。
- **手機被順手拿走**：可選的**本機密碼鎖**（PBKDF2 驗證，切到背景自動上鎖）避免他人開啟你的裝置就直接看到聊天。此為本機保護，**無法取代「連結即權限」**——持有連結者仍可解密。

### 金鑰如何不外洩
房間祕密是 32 bytes 隨機值，放在網址 fragment（`#` 之後）。依 W3C 規範，**瀏覽器不會把 fragment 送給伺服器**。由此祕密以 HKDF 衍生出三個互相獨立的值：

```
房間祕密(32B, 只在#之後)
      │  HKDF-SHA256（不同 info 標籤）
      ├── channelId    → 送伺服器做路由（無法解密任何東西）
      ├── AES-256-GCM  → 只在瀏覽器內加解密（永不離開裝置）
      └── 安全驗證碼    → 供成員當面核對，確認連結未被竄改
```

### ⚠️ 未在威脅模型內 / 使用者須知
- **你的裝置本身**：若在公司配發、裝有 MDM/端點監控的電腦上使用，對方可**側錄螢幕與鍵盤輸入**，這在裝置層級，任何 App（包含本工具）都擋不住。**真正的隱私請用個人裝置 + 個人網路。**
- **連結即權限**：任何拿到連結的人都能解密該房間。請透過**安全管道**分享，別貼在會被記錄/掃描的地方。連結會進入瀏覽器歷史紀錄。
- **前向保密（已實作）**：訊息內容**不再**用網址金鑰直接加密，而是走 Signal 式「群組寄件金鑰（sender keys）」：每位成員每次連線產生**一次性 ECDH 金鑰**，各自維護一條逐則前進的對稱棘輪，訊息金鑰用後即刪。網址金鑰只當「外層存取驗證」（決定誰能參與握手），**看得到密文也解不開內容**。因此即使隧道商／代理錄下密文、日後又取得連結，**過往訊息仍無法被回溯解密**；新加入者也只拿到當下金鑰，讀不到加入前的訊息。細節見 [public/js/session.js](public/js/session.js)。
- **尚未實作：連續的入侵後回復（post-compromise security）**：群組模式在「重新連線」時會換新棘輪根，但不像 1:1 Double Ratchet 那樣**逐則**做 DH 棘輪。若某成員裝置在單一連線期間被完全入侵，攻擊者可在該連線存續期間持續跟上；成對逐則 PCS 需 MLS/Double Ratchet 等更重的機制（未納入本版）。
- **成員身分**：房間內以自填暱稱顯示，非強身分驗證；房間完整性由「安全驗證碼」保證。

### 🚫 正確使用
本工具用於**保護個人私密通訊**。請**勿**用於外洩公司機密、規避法定合規稽核，或任何違反你所在組織政策/當地法律的用途。

## 🧱 架構

```
瀏覽器 A（PWA）                    Node 中繼伺服器                 瀏覽器 B（PWA）
──────────────                    ──────────────                ──────────────
明文 → AES-GCM 加密  ── 密文 ──►   只依 channelId 轉發   ── 密文 ──►  解密 → 明文
金鑰(來自#，不外傳)                 看不到金鑰/明文                    金鑰(來自同一條連結)
```

- 前端：純 HTML/CSS/ES Module，無框架、無 CDN、無第三方腳本 → 易審閱、供應鏈風險最小。
- 加密：瀏覽器內建 **Web Crypto API**（`crypto.subtle`），無自製密碼學原語。
- 後端：Node 內建 `http`/`crypto` 自行實作 **RFC 6455 WebSocket**，只做密文中繼，不落地儲存。

## 🧪 測試

```bash
npm test           # 加密自我測試（不需伺服器）：round-trip、竄改/錯誤金鑰拒收、IV 唯一性、channelId 格式
npm run test:fs    # 前向保密（不需伺服器）：多人收發、晚加入者讀不到舊訊息、亂序、ECDH 對稱、竄改拒收
npm run test:relay # 中繼行為（需先啟動 node server/server.js）：多人轉發、不回送自己、頻道隔離、惡意輸入不崩潰
```

測試檔位於 [tools/selftest.mjs](tools/selftest.mjs)、[tools/test-fs.mjs](tools/test-fs.mjs) 與 [tools/test-relay.mjs](tools/test-relay.mjs)。
